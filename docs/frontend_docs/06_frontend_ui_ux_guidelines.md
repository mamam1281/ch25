# 프론트엔드 UI/UX 가이드라인 (크리스마스 시즌)

- 문서 타입: 프론트엔드 UX 가이드
- 버전: v1.1
- 작성일: 2025-12-10
- 작성자: 시니어 백엔드/FE 협업 가이드
- 대상 독자: 프론트엔드 개발자, 디자이너, QA

## 1. 목적 (Purpose)
- 크리스마스 시즌 동안 User App의 톤앤매너, 색상, 인터랙션을 통일해 타겟(대한민국 20~60대 남성, 도박 경험자, 텔레그램 유입)에게 즉시 이해되는 경험을 제공한다.

## 2. 범위 (Scope)
- Scope: 모바일 우선 레이아웃, 색상/타이포/애니메이션 가이드, 페이지별 핵심 UI 패턴, CTA/피드백 구성, 타겟 적합성 체크.
- Out of Scope: 브랜드 로고/일러스트 제작, 실제 에셋 파일 관리, A/B 테스트 세부 설계.

## 3. 용어 정의 (Definitions)
- Primary Button: 페이지 내 최우선 액션 버튼(예: "오늘 룰렛 돌리기").
- Season Pass Bar: 상단 진행도 요약(레벨/XP/도장 표시).
- Feature Page: 오늘 활성화된 게임 화면(룰렛/주사위/복권/랭킹/레벨).

## 4. 본문 (가이드)
### 4-1. 기본 콘셉트 – 다크 카지노 + 크리스마스 악센트
- 배경: 어두운 톤 (`bg-slate-900`, `bg-neutral-950`), 텍스트는 고대비 화이트/그레이(`text-white`, `text-slate-200`).
- 포인트 컬러: Primary는 네온 그린(`#22c55e` 근처), Secondary는 레드(`#ef4444`)와 골드(`#facc15`).
- 적용 규칙:
  - CTA(스핀/시작/긁기) → 네온 그린.
  - 긴박/한정(오늘만, D-3, 마감 임박) → 레드/골드 배지나 텍스트.
  - 배경 그래디언트와 눈/전구는 정보 가독성을 가리지 않는 선에서만 사용.

### 4-2. 공통 컴포넌트 스타일 (User App)
- 카드: `bg-slate-900/80` + `backdrop-blur`, `rounded-2xl`, `p-4~p-6`, `shadow-lg`.
- 메인 CTA 버튼: `class="w-full py-3 rounded-full bg-emerald-500 hover:bg-emerald-400 text-white font-semibold text-lg"`; 필요 시 🎄 🎁 ⭐ 아이콘을 소량 사용.
- Season Pass Bar: `bg-slate-800` 바에 그린 진행도 + 눈송이 아이콘, 우측에 "Lv.x / 다음 보상까지 XP y" 텍스트.
- 상태 피드백: 클릭 → 로딩(disabled+스피너) → 결과 모달/토스트. 실패 시 재시도 버튼 제공.

### 4-3. 페이지별 UI 기조
- `/roulette`: "오늘의 메인 이벤트" 톤. 상단 타이틀(🎄 크리스마스 룰렛) + "오늘은 1번만" 서브텍스트. 중앙 룰렛 휠(그린/레드 악센트), 하단 참여 가능 여부 + CTA + 레벨 도장 안내.
- `/dice`: 배틀 느낌. 두 주사위 애니메이션이 중앙, 결과 카드는 크게, "오늘 남은 기회 n회"는 레드 배지로 CTA 근처에 표시.
- `/lottery`: 긁기/오픈 경험. 큰 복권 카드나 선물 상자 한 개, 탭/클릭으로 오픈 후 모달로 결과 노출. 재고/확률 안내는 보조 텍스트.
- `/ranking`: Top10 리스트 + 내 위치 하이라이트. 그래프 없이 카드 리스트 + 강조만. 데이터 없음 시 빈 상태 문구.
- `/season-pass`: 진행도/레벨/도장/보상 상태를 한 눈에. Stamp 가능 시 CTA를 상단에 노출, 레벨별 보상은 카드/아코디언.

### 4-4. 타겟(20~60대 남성, 도박 익숙) 맞춤 기준
- 카툰/어린이풍 금지, 성인 카지노 톤 유지. 캐릭터 사용 시 성인 남/여 실루엣 정도로 제한.
- 가독성 우선: 중요 숫자/날짜/보상은 `text-lg` 이상, 고대비 색상 유지.
- 선택 최소화: 주요 행동 버튼 1~2개로 제한(예: "돌리기" vs "나중에").
- 즉시성: 스핀/굴림/긁기 후 1~2초 내 결과 모달. 로딩만 길게 노출 금지.
- 손실 프레이밍 최소화: 실패/마감 안내와 동시에 "내일 다시", "레벨 진행도" 등 다음 행동을 제시.

## 5. 예시
- CTA 클래스 예시: `className="w-full py-3 rounded-full bg-emerald-500 hover:bg-emerald-400 text-white font-semibold text-lg"`
- 오류 토스트 예시: `showToast({ type: 'error', message: '다시 시도해주세요' })`

## 6. Validation / Checklist
- [ ] 모바일 우선/데스크톱 우선 구분이 명시되었는가?
- [ ] 색상 팔레트가 다크 + 그린/레드/골드 악센트 조합으로 정의되었는가?
- [ ] CTA가 스크롤 없이 1차 가시 영역에 위치하도록 가이드했는가?
- [ ] 로딩/에러/성공 패턴과 Season Pass Bar 표시 규칙이 정의되었는가?
- [ ] 접근성 고려(포커스, aria-label)가 언급되었는가?

## 7. UI 타겟 적합성 검증 체크리스트
- [ ] 전체 색감이 '다크 + 크리스마스 악센트(그린/레드/골드)' 조합으로 유지되는가?
- [ ] 주요 CTA(스핀/주사위/복권/스탬프)는 접속 시 스크롤 없이 보이는 영역에 위치하는가?
- [ ] 텍스트/숫자는 20~60대 남성이 모바일에서 쉽게 읽을 크기(중요 정보 `text-lg` 이상)로 설정되었는가?
- [ ] 일러스트/아이콘은 성인 카지노/크리스마스 장식 수준이며 어린이풍 요소가 없는가?
- [ ] 각 화면의 선택지는 1~2개로 제한되어 있는가?
- [ ] 이벤트 참여 → 결과 확인까지 3스텝 이내로 끝나는가?
- [ ] 결과 화면이 손실만 강조하지 않고, 다음 행동(내일 재참여, 레벨 안내)을 함께 제공하는가?
