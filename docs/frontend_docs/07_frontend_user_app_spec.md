# 유저 앱 화면 스펙

- 문서 타입: 프론트엔드 사용자 앱 스펙
- 버전: v1.1
- 작성일: 2025-12-10
- 작성자: 시니어 백엔드/FE 협업 가이드
- 대상 독자: 프론트엔드 개발자, QA, PM

## 1. 목적 (Purpose)
- 유저용 이벤트 SPA의 페이지별 구성, 호출 API, 상태/에러 처리, 크리스마스 시즌 UI 기준을 명세한다.

## 2. 범위 (Scope)
- Scope: `/`, `/roulette`, `/dice`, `/lottery`, `/ranking`, `/season-pass` 화면의 UI/데이터 흐름, CTA/피드백, 타겟(20~60대 남성) 적합 UI 체크.
- Out of Scope: 관리자 화면, 백엔드 로직.

## 3. 용어 정의 (Definitions)
- Feature Page: 오늘 활성화된 이벤트 페이지(룰렛/주사위/복권/랭킹).
- Season Pass Bar: 상단 요약 영역(레벨/XP/도장 수) 표시.

## 4. 본문 (페이지 스펙)
- `/` 홈: `useTodayFeature` 호출 → feature_type에 맞춰 자동 이동. 로딩 시 스피너, 에러 시 재시도/공지.
- `/roulette`: status 조회(남은 스핀) → 룰렛 휠 애니메이션(그린/레드) → 결과 모달. CTA는 1차 가시 영역에 배치, 레벨 결과 병행 표시.
- `/dice`: status 조회(남은 회수) → 두 주사위 애니메이션 → 합계 비교 결과/보상 카드. 레드 배지로 남은 회수 강조.
- `/lottery`: status 조회(티켓, prize 프리뷰) → 복권 카드/선물 상자 오픈 → 당첨/꽝 모달. 재시도/내일 안내 텍스트.
- `/ranking`: today 랭킹 리스트 + 내 순위 하이라이트. 데이터 없음 시 빈 상태 문구.
- `/season-pass`: 현재 시즌 정보, 진행도, 레벨별 보상 상태, stamp 버튼(가능 시) 표기. 진행도 바는 상단 고정.
- 공통: 로딩/에러/빈 상태, 인증 실패(401) 처리, SeasonPass 연동 응답 표시 규칙.

## 5. 예시
- `/dice` 결과 JSON 사용 예:
```json
{
  "game": {
    "user_dice": [4, 6],
    "dealer_dice": [3, 2],
    "user_sum": 10,
    "dealer_sum": 5,
    "outcome": "WIN",
    "reward_type": "POINT",
    "reward_amount": 10000
  },
  "season_pass": {"added_stamp": 1, "xp_added": 50}
}
```

## 6. Validation / Checklist
- [ ] 모든 페이지가 호출하는 API와 응답 필드가 명확히 표기되었는가?
- [ ] today-feature 없는 날의 처리(공지/빈 상태)가 정의되었는가?
- [ ] SeasonPass 바/결과 표기 위치가 일관되게 명시되었는가?
- [ ] 로딩/에러/빈 상태 UI 요구사항이 포함되었는가?

## 7. 페이지별 UI/UX 검증 체크리스트
### `/roulette`
- [ ] 상단에 '오늘의 크리스마스 룰렛' 타이틀과 오늘 참여 조건이 한 눈에 보이는가?
- [ ] 룰렛 휠이 모바일 화면 가로 폭 70~90%를 차지하며 터치 영역이 충분한가?
- [ ] "오늘 남은 기회" 문구가 CTA 근처에 노출되는가?
- [ ] CTA 텍스트가 행동 유도형(예: "룰렛 돌리기")이고 1차 가시 영역에 있는가?
- [ ] 결과 모달이 당첨/꽝과 함께 레벨 진행도나 내일 안내를 보여주는가?

### `/dice`
- [ ] 두 주사위 애니메이션이 명확히 보이며 결과 카드가 큰 글꼴로 표시되는가?
- [ ] 남은 회수를 레드 배지로 강조하고 CTA 근처에 배치했는가?
- [ ] 승/무/패 결과와 보상이 한 화면에서 즉시 확인되는가?

### `/lottery`
- [ ] 큰 복권 카드/선물 상자가 중앙에 배치되어 한 번의 탭/클릭으로 오픈되는가?
- [ ] 당첨/꽝 결과 모달이 즉시 노출되고, 내일/다음 기회 안내를 포함하는가?
- [ ] 상품/확률 안내가 보조 텍스트로 과하지 않게 배치되었는가?

### `/ranking`
- [ ] Top 리스트와 내 순위가 명확히 구분되어 하이라이트되는가?
- [ ] 데이터 없음 시 빈 상태 메시지가 제공되는가?

### `/season-pass`
- [ ] 진행도 바가 상단에 고정되고 레벨/XP/도장 정보가 한 번에 보이는가?
- [ ] Stamp 가능한 경우 CTA가 상단 영역에서 바로 보이는가?
- [ ] 레벨별 보상 상태가 카드/아코디언으로 가독성 있게 정렬되었는가?
