# 운영 문서 온보딩 가이드 (실제 적용용)

목적: 현재 폴더(`docs/06_ops/202601/`)에 흩어진 운영 문서를 **현장 적용 순서**로 재정렬해서, 운영자가 “오늘 뭘 해야 하지?”를 바로 실행할 수 있게 만든다.

---

## 0) 1분 요약(운영 실행 순서)
1) **원칙 고정**: 자동=티켓 / 수동=현금성(포인트·코인·키)
2) **기간 선택**: (브릿지 주간/마감/공백/재개) 중 지금이 어디인지 먼저 결정
3) **레버 조작**: Vault 배율 + 룰렛/복권 보상 테이블 + 카피(배너/모달/공지)
4) **리스크 가드레일**: XP 폭주/수동지급 폭주/경제 인플레를 먼저 막기
5) **측정**: “충전(결제) KPI”는 엑셀로, “행동 퍼널”은 별도 이벤트 로그가 있어야 함

## 1) 문서별 ‘언제 열고 무엇을 적용’하는가

### A. 전체 운영의 기준점(가장 먼저 보는 문서)
- `20251226_master_operation_plan_retention_cc.md`
  - 역할: **통합본(우선순위/원칙/기간별 운영/가드레일)**
  - 실제 적용: 운영 결정을 내릴 때(“배율을 올릴까?”, “보상을 바꿀까?”) **항상 이 문서의 원칙(수동지급/손실회피/연속성/XP 가드레일)**을 먼저 체크

### B. ‘언제 무엇을 한다’ 캘린더(기간 운영)
- `20251225_operation_plan_bridge_week.md`
  - 역할: **12/25~1/1 브릿지 주간 타임라인 + 지급 정책 + 백업/초기화**
  - 실제 적용: 오늘 날짜가 브릿지 주간이면 이 문서대로 **기간별 액션(2배→마감→백업→공백공지)** 실행

### C. 공백/장애 대비 런북(최소 동작)
- `202512ops_gap_playbook_2025-12-25.md`
  - 역할: **1/1~1/2, 1/6~1/7 공백일에 “망하지 않게” 유지하는 최소 체크리스트**
  - 실제 적용: 공백일/장애 모드에서는 “기능 추가”보다 **안내/기대치 관리/백업/로그 확인**이 전부

### D. 실전 방어 운영(체리피커/꽁머니 유저)
- `202601_operation_guide_cherrypicker_defense.md`
  - 역할: **미끼(티켓) 무제한 + 문(레벨7+) 수동 지급**으로 비용 폭탄을 막는 실전 운영
  - 실제 적용: 유입이 텔레그램 중심이고 ‘꽁머니 유저’가 몰리는 구간에, L7+ 보상은 무조건 수동 심사로 유지

### E. 장기 설계/연속성(주간 인플레 방지)
- `2025season_cycle_efficiency.md`
  - 역할: **주간 리셋 피로/인플레 문제를 “누적형 자산 흐름(주→월→분기)”로 바꾸는 방향성**
  - 실제 적용: 단기 이벤트(브릿지)만 반복하면 내성 생김 → 월간/분기 앵커(4주/12주)를 UI/카피에 반드시 연결

### F. KPI/데이터 진실성(엑셀 기반)
- `excel_data_audit_CC1229_ko.md`
  - 역할: 제공된 엑셀로 **무엇을 측정할 수 있고(충전 KPI), 무엇은 불가능한지(12/30~1/1, 행동로그)** 경계 정의
  - 실제 적용: “측정 가능한 것”만 KPI로 내고, 측정 불가 항목은 **추가 데이터 요구 스펙**부터 확정

## 2) 운영자가 실제로 조작하는 ‘레버(Levers)’ 맵

### 2.1 Vault(금고) 레버
- `VAULT_ACCRUAL_MULTIPLIER`
  - 사용 시점: 12/25~12/27 가속(2.0), 이후 원복(1.0)
  - 주의: 배율 상향은 재방문/체류엔 좋지만, **자산 폭주/수동지급 기대치 상승**을 같이 키움

### 2.2 룰렛/복권(티켓 연료) 레버
- 어드민에서 교체하는 대상
  - 룰렛: `roulette_config` + `roulette_segment(6칸)`
  - 복권: `lottery_config` + `lottery_prize`
- 원칙
  - Phase 1(유입 초반)에는 **원가 0 티켓** 중심으로 “혜자 체감”만 만든다
  - 현금성(`POINT/CC_COIN/Key`)은 **수동 지급 구간과 연결**되므로 초반 자동 지급에 넣지 않는다

### 2.3 시즌패스/수동지급 레버
- 레벨 7~10은 “수동 지급”이므로 운영이 감당 가능한 속도로만 레벨업이 일어나야 한다
- XP 관련 가드레일(핵심)
  - XP 상향 금지(대신 제한형 부스터)
  - 룰렛/복권/티켓 소모에 XP를 붙이지 않기
  - 필요 시 일일 XP 상한/soft cap

### 2.4 카피/배너/공지 레버(운영 공백 포함)
- 공백일(1/1~1/2, 1/6~1/7)
  - “정산/점검 중” + “수동 지급” + “재개 시각” 3요소를 항상 포함
- 손실 회피 카피
  - “보너스”가 아니라 “내 자산이 소멸/락업될 수 있음” 프레임

## 3) 실전 적용 템플릿(상황별 체크리스트)

### 3.1 브릿지 주간(12/25~12/31) 운영 체크리스트
- (가속 구간) 12/25~12/27
  - [ ] `VAULT_ACCRUAL_MULTIPLIER=2.0`
  - [ ] 배너: “올해 마지막 2배 적립”
  - [ ] 룰렛/복권: 티켓 중심(현금성 제외)

- (마감 압박) 12/28~12/30
  - [ ] 배율 원복(원칙 1.0) 또는 지표 보고 유지
  - [ ] 타겟 메시지: L7 미달/Gold Key까지 2단계 남음 등

- (결산) 12/31
  - [ ] 23:50 DB 백업
  - [ ] 23:59 시즌 종료/스냅샷/초기화(정의된 범위)

### 3.2 운영 공백(1/1~1/2, 1/6~1/7) 체크리스트
- [ ] 상단 배너/공지: 정지 기간 + 재개 시각 명시
- [ ] 금고/보상 UI 카피: “수동 지급” 기대치 고정
- [ ] 로그 확인 경로 공유(backend/frontend)
- [ ] (가능하면) 최소 모니터링 1회: 안내가 실제로 노출되는지

### 3.3 체리피커 방어(유입 폭주 시) 체크리스트
- [ ] L1~6: 티켓은 무제한(미끼)
- [ ] L7+: 포인트/코인/키는 100% 수동 심사(문)
- [ ] Vault는 “잠긴 자산”으로 상실회피를 강화(덫)
- [ ] 수동지급 대기건수 폭주 시: 배율 하향/확률 조정/공지로 지연을 선제 고지

## 4) 측정(엑셀 기준) — 무엇이 ‘증명 가능한 KPI’인가

- `CC1229업데이트_완료.xlsx` 기반으로 확실히 할 수 있는 것
  - 일별 충전건수, 일별 유니크 충전자
  - 세그먼트(리텐션그룹/행동유형)별 충전 기여

- 이 파일만으로는 할 수 없는 것
  - 12/30~1/1 기간 분석(데이터 없음)
  - 로그인/플레이/미션 등 행동 퍼널/DAU/리텐션(이벤트 로그 필요)

## 5) 운영자가 매일 보는 최소 대시보드(권장)
