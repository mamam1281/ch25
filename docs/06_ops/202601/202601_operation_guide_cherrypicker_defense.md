---

## 1. 핵심 전략: "미끼는 싸게, 문은 단단하게"

운영의 핵심은 **내부 재화(티켓)**와 **외부 재화(포인트/코인)**를 철저히 분리하는 것입니다.

### 1-1. 미끼 (Bait): Level 1 ~ 6
*   **지급 품목**: 룰렛 티켓, 다이스 티켓, 번들 (내부 재화, 원가 0원).
*   **운영 원칙**: **무제한 지급**. 유저가 "혜자 사이트"라고 느끼게 만들어 게임판에 묶어둡니다.
*   **효과**: 티켓 소모 → 금고(Vault) 잔액 증가 → "내 돈이 쌓였다"는 착각(Loss Aversion) 유발.

### 1-2. 덫 (Trap): Vault (금고)
*   **역할**: 유저의 활동(배팅)을 '잠긴 자산'으로 시각화.
*   **심리 전술**: "지금 멈추면 이 돈이 사라집니다"라는 메시지로 재방문과 입금을 유도.

### 1-3. 문 (Gate): Level 7 이상
*   **지급 품목**: 포인트, 코인(값비싼 외부 재화), Gold/Diamond Key.
*   **운영 원칙**: **100% 수동 지급 (Manual Review)**.
*   **심사 기준**: 입금 내역 확인, 다계정 여부 확인. 진성 유저만 통과시킵니다.

## 3. 운영 모니터링 (SQL 가이드)

DB가 초기화되었으므로, **지금부터 쌓이는 데이터**를 실시간으로 감시해야 합니다. (6시간 간격 권장)

### 3-1. 실시간 지급 현황 조회 (비용 감시)
```sql
-- 1. 티켓 지급량 (활동성 지표): 많을수록 좋음
SELECT 'Total Tickets (Activity)' as type, SUM(amount) 
FROM user_game_wallet_ledger 
WHERE amount > 0;

-- 2. 포인트/코인 지급량 (비용 지표): 급증하면 경계해야 함
SELECT 'Total Points (Cost)' as type, SUM(amount) 
FROM user_cash_ledger 
WHERE amount > 0;
```

### 3-2. 고액 자산 보유자 감시 (잠재적 폭탄)
```sql
-- 잠긴 금고 잔액이 50만 포인트 이상인 유저 조회
SELECT user_id, locked_balance 
FROM vault_status 
WHERE locked_balance >= 500000 
ORDER BY locked_balance DESC 
LIMIT 10;
```
