# Documentation Rules (v1.0)

이 문서는 docs/ 내 모든 문서를 작성·갱신할 때 따라야 할 공통 규칙을 정의합니다.

## 1. 파일/이름/언어
- 이름: `NN_section_title_vX.Y.md` 형태 권장(예: `05_module_roulette_service_v1.0.md`). 단일 가이드/런북은 명확한 slug 사용(예: `LOCAL_DEVELOPMENT.md`).
- 언어: 팀 기본은 한국어, 코드/식별자는 영어 유지. 혼용 시 문장/표 설명은 한국어, 코드/필드명은 원문 그대로.
- 문자: ASCII 위주. 불가피한 경우에만 한글/특수문자 사용.

## 2. 상단 메타 블록
문서 첫 부분에 다음 메타를 적습니다(한 줄에 하나).
- 문서 타입: 설계 / API / DB / 모듈 / 가이드 / 런북 / 체크리스트 / 회고 / 변경 로그
- 버전: `vX.Y` (예: v1.0)
- 작성일: YYYY-MM-DD
- 작성자: 팀/작성자명
- 대상 독자: 예) BE 개발자, FE 개발자, SRE, PO 등

## 3. 구조/섹션 기본 틀
- 개요 섹션: 목적(Purpose), 범위(Scope), 용어 정의(Definitions)로 시작.
- 본문 섹션: 기능/정책/흐름을 번호 기반 제목(`## 1. ...`)으로 정리.
- 변경 이력: 문서 하단에 역순으로 기록.
- 체크리스트/런북: 번호/체크박스 사용, 실행 명령은 fenced code block.

## 4. 제목/번호 규칙
- H1은 문서 제목 하나만. 이후 H2부터 번호를 붙입니다(예: `## 1. 목적`).
- 목록: 불릿은 `-`, 단계는 `1)` 형태. 동일 레벨에서 혼용 금지.

## 5. 링크/참조
- 내부 문서 링크는 상대경로 사용: `docs/03_api/03_api_overview_v1.0.md`.
- 코드/파일 참조 시 경로를 명시: `app/services/vault_service.py`.
- 외부 링크는 명확한 라벨과 URL 함께 기재.

## 6. 표/코드/예시
- 표는 헤더 포함. 숫자 단위/통화/시간대(KST/UTC)를 명시.
- 코드/명령/SQL은 fenced code block + 언어 태그 사용: ```sql ...``` / ```bash ...```.
- JSON 예시는 최소 키만 포함, 주석은 `//` 대신 별도 설명문.

## 7. 용어/포맷 일관성
- 시간/시간대: ISO 8601(예: `2025-12-25T12:00:00+09:00`), 날짜는 `YYYY-MM-DD`.
- 금액/포인트: 쉼표 없이 정수(예: 10000). 단위는 문장에 병기.
- Boolean 표기: TRUE/FALSE 대신 `true/false` 소문자.
- 에러 코드/상수는 UPPER_SNAKE_CASE 그대로 표기.

## 8. 상태/검증/QA 섹션
- 기능 문서에는 "운영상 주의" 또는 "QA/검증" 섹션을 포함.
- 체크리스트는 `[ ]` 미완 / `[x]` 완료 표기, 테스트 경로/명령을 함께 적기.

## 9. 그림/다이어그램
- 가능하면 텍스트 다이어그램 사용(mermaid 가능 시 fenced `mermaid`).
- 이미지 파일은 `docs/img/` 하위에 두고 캡션/출처 기재.

## 10. 변경/버전 관리
- 내용 변경 시 버전 증가(마이너: 문구/예시, 메이저: 정책/스펙 변경).
- 변경 이력 섹션 예시:
  - v1.1 (2025-12-25, BE팀): 금고 해금 구간 A/B 추가
  - v1.0 (2025-12-20, BE팀): 최초 작성

## 11. 금지/주의
- 미확정 정책은 "제안"으로 명시하고 기본값과 다르게 동작하면 강조.
- 중복 문서 금지: 기존 문서와 겹치면 기존 문서에 섹션을 추가하거나 최신본에 합칩니다.
- 모호한 표현(“적당히”, “etc”) 지양, 수치/조건/예외를 구체화.

## 12. 리뷰 체크리스트(작성자가 셀프체크)
- [ ] 메타 블록 작성(타입/버전/날짜/작성자/대상)
- [ ] 목적/범위/용어 정의 포함
- [ ] 번호 붙은 섹션 구조 유지
- [ ] 코드/명령/SQL에 fenced block 사용
- [ ] 경로/파일/엔드포인트 정확히 표기
- [ ] 변경 이력 갱신
- [ ] 정책/수치/예외가 모호하지 않음

## 13. 배치 위치 기준
- 설계/모듈: `docs/05_modules/`
- API: `docs/03_api/`
- DB: `docs/04_db/`
- 가이드/런북/체크리스트/운영: `docs/06_ops/` 또는 상위 루트 `docs/`
- 아카이브: 폐기/구버전은 `docs/_archive/`에 이동 후 최신본 링크를 남김
