# XMAS Event System 외부 AI 브리프
문서 타입: 가이드
버전: v1.0
작성일: 2025-12-25
작성자: 시스템 설계팀
대상 독자: 마케팅 전문가, 리텐션 행동기획가, 행동경제학 전문가, 광고 카피라이터

## 1. 목적 (Purpose)
- 외부 AI가 프로젝트, 기능, 리텐션 요소를 정확히 이해하고 실질적인 개선 아이디어를 제안할 수 있도록 핵심 정보를 단일 문서로 제공한다.
- 마케팅/리텐션/행동경제학/카피라이팅 관점의 개선안을 빠르게 수렴하기 위한 브리프를 제공한다.

## 2. 범위 (Scope)
- 포함: 현재 운영 중인 기능, 리텐션 루프, 사용자 행동 특성, 카피 노출 지점, 제약 조건.
- 제외: 코드 수준 구현 세부, 인프라/배포 절차, DB 스키마 상세.

## 3. 용어 정의 (Definitions)
- Event System: CC Casino 유저를 대상으로 한 웹 기반 미니게임/시즌패스/랭킹 이벤트 플랫폼.
- Token: 게임 플레이에 필요한 소비성 재화(ROULETTE_COIN, DICE_TOKEN, LOTTERY_TICKET).
- Ticket Zero: token_balance=0 상태에서 플레이가 막히는 구간. 즉시 행동(CTA) 전환이 핵심.
- Season Pass: XP/레벨 기반 진행도와 보상 체계.
- Vault: 잠긴 잔액(locked balance) 기반의 손실 회피 리텐션 장치.
- Team Battle: 팀 간 점수 경쟁 기반 리텐션 기능.
- External Ranking: 운영자가 수동 입력하는 외부 랭킹 및 보상 체계.
- Survey: 설문 참여 보상으로 재접속을 유도하는 리텐션 모듈.
- Event Bridge: 시즌 종료 전후의 공백 구간을 메우는 보상 예약/카운트다운 흐름.

## 4. 프로젝트 요약 (Project Summary)
- CC Casino 유저의 재방문과 체류시간을 늘리기 위한 **미니게임 기반 리텐션 플랫폼**이다.
- 핵심 목표는 짧은 플레이 루프 + 진행/경쟁/손실 회피 요소로 습관형 사용을 만드는 것이다.
- 과거 Daily Feature(today-feature) 게이트는 폐기되었고, 현재는 **항상 열려있는 게임/시즌패스 중심 구조**다.

## 5. 타겟 유저 & 심리
- 핵심 타겟: 한국 20~40대 남성(확장 타겟 20~60대).
- 특성:
  - 빠른 결과와 즉각적 피드백 선호, 대기/설명 장벽을 싫어함.
  - 경쟁/랭킹/팀 결과에 민감.
  - 손실 회피와 보유 효과에 반응(잠긴 자산, 만료 경고).
  - 다크 카지노 톤의 시각적 자극을 선호(아동풍 금지).

## 6. 핵심 기능/모듈 (현재 기준)
| 기능 | 상태 | 요약 | 리텐션 역할 |
| --- | --- | --- | --- |
| 미니게임 3종 | 운영 | 룰렛/주사위/복권 즉시 결과형 게임 | 빠른 도파민 루프 |
| 시즌 패스/레벨 | 운영 | 일일 스탬프/XP/레벨 보상 | 장기 목표/누적 투자 |
| 팀 배틀 | 운영 | 2팀 경쟁, 게임 플레이가 팀 점수로 전환 | 경쟁심/소속감 |
| 외부 랭킹 | 운영 | 운영자 수동 입력/보상 | 과시/성취 |
| 게임 토큰 지갑 | 운영 | token_balance 기반 플레이 가능 여부 | 희소성/전환 트리거 |
| Vault Phase 1 | 운영 | locked balance 단일 기준, unlock 시 보상 지급 | 손실 회피 |
| 신규회원 주사위 | 폐기 | 1회 판정, 패배 시 locked balance 최소 10000 보정 | 온보딩/잠금 자산 각인 |
| Survey 리텐션 | 구현/운영 가능 | 설문 참여 보상, 트리거(레벨업/미접속/게임결과/수동) | 복귀 유도 |
| Event Bridge | 기간 한정 | 키 수집/보상 예약, 시즌 공백 연결 | D7 유지 |
| Daily Feature | 폐기 | today-feature 게이트 제거(404/410) | 사용 안 함 |

## 7. 리텐션 루프/요소
### 7-1. 플레이 루프
- Token 소비 → 게임 결과 → XP/보상 → 시즌 패스 진행 → 추가 플레이 동기 형성.

### 7-2. 경쟁/소셜 루프
- 팀 배틀 점수와 리더보드로 경쟁 심리 자극.
- 인앱 토스트/배너 중심 알림(OS 푸시는 현재 없음).

### 7-3. 손실 회피 루프
- Vault의 잠긴 잔액을 “내 자산”으로 인지시키고 소멸/만료 프레임으로 행동을 촉진.
- Ticket Zero 구간에서 금고 CTA로 전환시키는 것이 핵심.

### 7-4. 온보딩/초기 리텐션
- 신규회원 주사위 1회 판정으로 “임시 금고” 메시지를 강제 노출.
- 첫 경험에서 “내 자산이 잠겨있다”는 인식을 심는 것이 목적.

### 7-5. 설문 기반 리텐션
- 설문 참여 시 토큰/포인트 지급으로 복귀 동기 제공.
- 트리거: LEVEL_UP, INACTIVE_DAYS, GAME_RESULT, MANUAL_PUSH.

### 7-6. 시즌 브리지
- 시즌 종료 전후 공백을 키 수집/보상 예약으로 연결해 리텐션 유지.

## 8. 카피 노출 지점 & 톤 가이드
| 위치 | 트리거 | 목적 | 톤 가이드 |
| --- | --- | --- | --- |
| 홈 카드/게임 진입 | 기본 진입 | 다음 행동 제시 | 짧고 직설적, 1~2개 CTA |
| Ticket Zero 모달 | token_balance=0 | 이탈 방지/전환 | “내 자산 회수” 프레임 |
| 시즌 패스 임박 | 다음 레벨 80~90% | 매몰 비용 자극 | 수치/잔여량 강조 |
| 팀 배틀 열세/역전 | 점수 차 임계치 | 경쟁 심리 자극 | “우리 팀” 1인칭 |
| Vault 상태 | locked/만료 임박 | 손실 회피 | “소멸/보관 중/남은 시간” |
| 설문 배너 | 트리거 매칭 | 복귀/정보 수집 | 간결 + 즉시 보상 명시 |

### 8-1. 금고 카피 금지/지향
- 지양: “보상”, “이벤트”, “참여”, “충전하세요”
- 지향: “보관 중”, “소멸 예정”, “남은 시간”, “내 자산”

## 9. 현재 문제/기회
- Ticket Zero에서 “문의” 중심의 수동 흐름이 이탈을 유발.
- 온보딩 안내 부족으로 첫 행동이 느리고 아하 모먼트가 지연됨.
- 외부 트리거(푸시/DM) 부재로 복귀 자극이 약함.
- 팀 배틀의 즉각 피드백(기여/역전)이 체감상 부족.
- Vault 3단계(골드/플래티넘/다이아) 전략은 설계만 존재하며 UI/로직 미완.

## 10. 측정/지표 (권장)
- D1/D7 리텐션, 일간 활성 사용자(DAU), 평균 플레이 수/일.
- Ticket Zero 진입 대비 CTA 클릭/전환율.
- 시즌 패스 레벨업 전환율, 수동 보상 클레임율.
- 팀 배틀 참여율, 열세/역전 알림 후 재접속률.
- Vault unlock 전환율, 만료 직전 재방문율.
- 설문 참여율 및 완료율.

## 11. 제약/운영 조건
- 채널: OS 푸시 미구현, 인앱 배너/토스트/모달 중심.
- today-feature 게이트 폐기 상태 유지(항상-on 구조).
- 금액/포인트 표기는 쉼표 없이 정수(예: 10000원).
- 모든 날짜/시간은 KST 기준.
- 고가 보상은 수동 클레임 비중이 높음(즉시 지급 어려움).

## 12. 외부 AI에게 요청하는 산출물
- 리텐션 개선 아이디어 5~10개: 기대 효과, 적용 위치, 난이도(낮음/중간/높음) 포함.
- 카피라이팅 제안: Ticket Zero, Vault, 시즌 패스 임박, 팀 배틀 열세/역전, 설문 배너 문구.
- 행동경제학 기반 가설: 어떤 편향을 어떤 화면/행동에 적용할지 명시.
- A/B 테스트 제안: 지표, 실험 기간, 성공 기준 포함.
