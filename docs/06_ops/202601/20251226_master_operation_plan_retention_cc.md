- **Asset Grand Cycle(1:4:12)** 프레임으로 “내 돈(자산)이 쌓여서 사라질 수 있다”를 각 화면/알림/보상에 일관되게 녹이고,

## 1) 운영 목표 / 우선순위
### 1.1 최우선 목표
1) **리텐션 유도**: “안 하면 손해” 프레임(손실 회피)으로 재방문을 강제한다.
2) **외부 CC 사이트 이용 유도**: 티켓 소진/마감 압박/금고(지갑) 확인 흐름을 통해 **충전→보너스 티켓→추가 플레이** 루프를 만든다.

### 1.2 보조 목표
- 경제 인플레 방어: 현금성(포인트/코인/키)은 **수동 지급**으로 통제.
- 운영 공백 대비: 서버가 멈춰도 “공지/배너/약속된 지급일”은 유지.

### 1.3 명시적 비목표(Non-goals)
- 공백 기간 동안 “게임 내 자동 현금성 지급” 구현.
- 보상 총량을 키우는 방향의 즉흥 상향.

## 2) 핵심 원칙(문서 공통 정렬)
1) **지급 차별화**
- 자동 지급: 내부 재화(룰렛/다이스/복권 티켓, 번들)
- 수동 지급(관리자): 외부 재화(포인트/코인), Key(Gold/Diamond), 1만 포인트 이상 현금성 보상

2) **손실 회피(Loss Aversion)**
- 카피/배너/푸시에서 “보상”이 아니라 “이미 쌓인 내 자산”이 **소멸/락업**될 수 있다는 압박을 유지.

3) **연속성(주간→월간→분기)**
- 1주(시즌패스 마감) / 4주(월간 금고) / 12주(대금고) 구조로 “이번 주 성과가 다음 단계로 이어진다”를 반복.

4) **운영 리스크 최소화**
- 공백일에는 자동 지급 기대를 차단하고, 지급일/방식(수동)을 강하게 고지.

## 3) 시스템 구조(운영자가 조작 가능한 레버)
### 3.1 Vault(금고/코드지갑) 구조
- 플레이 기반 적립(예: 판당 기본 적립 + 조건부 보너스)
- 이벤트 기간에는 적립 배율(Multiplier)로 속도 조절

운영 레버(개념):
- `VAULT_ACCRUAL_MULTIPLIER` (예: 2.0 → 1.0)
- (선택) 패배 보너스/추가 적립 같은 “판수 단축 장치”

핵심 운영 포인트:
- 12/25~12/27: **적립 2배**로 “올해 마지막 가속” 분위기
- 12/28 00:00: 원칙적으로 **1.0으로 원복**(또는 지표 보고 유지)

### 3.2 시즌패스(레벨/보상/수동지급)
- 브릿지 주간 핵심 목표: **L10(Diamond Key)까지 완주**
- L7~L10은 수동지급 구간이므로 운영폭주(지급 대기건수) 리스크가 큼

### 3.3 룰렛/복권(무한 루프 연료)
- 목적: 플레이 연료(티켓)를 공급해 체류를 늘리고, Ticket Zero 순간에 외부 CC로 연결
- 운영 레버(개념):
  - 룰렛: `roulette_config`(활성/일일 회수) + `roulette_segment`(6칸 보상/가중치)
  - 복권: `lottery_config`(활성/일일 티켓) + `lottery_prize`(보상/가중치/재고)

주의:
- “티켓이 티켓을 낳는” 루프는 리텐션에는 좋지만 경제/레벨링을 폭주시키기 쉬움
- 따라서 XP/레벨링은 이 루프와 **분리**(아래 XP 리스크 섹션)

## 4) XP 과다 지급 리스크(중요) + 운영 가드레일
### 4.1 왜 위험한가
- 이 구간은 L7~L10이 **수동지급(포인트/키)**에 연결된다.
- XP를 과하게 주면:
  - 레벨이 1~2일 만에 급상승 → 수동지급 대상자 급증(운영/비용 리스크)
  - “Next Season XP 부스터(첫 3미션 ×2)” 같은 설계와 **곱연산**으로 폭주 가능
  - 티켓 루프(룰렛/복권)까지 XP에 연결되면 “티켓→반복→XP→레벨업” 가속 루프 발생

### 4.2 운영 가드레일(권장)
- **원칙 A: XP 상향 금지, 대신 ‘제한된 부스터’만 사용**
  - 예: “첫 3개 미션만 ×2”처럼 횟수 제한이 있는 형태
- **원칙 B: XP는 게임/미션에서만, 룰렛/복권/티켓소모에는 XP를 붙이지 않기**
- **원칙 C: 일일 XP 상한 또는 구간 감쇠(soft cap)**
  - 목표: 하루에 레벨이 ‘너무 많이’ 오르지 않게(운영폭주 방지)
- **원칙 D: L7~L10 보상 수령에 ‘일수 게이트’(선택)**
  - 예: “서로 다른 날짜 2~3일 플레이” 같은 완충을 보상 수령 조건으로 추가하면
    XP 폭주에도 수동지급 폭주를 막을 수 있음

운영 체크(필수 질문 → 설정 확정용):
- 레벨별 필요 XP 곡선(레벨업 난이도)
- XP 적립 소스(게임 1판/미션/룰렛/복권 등)
