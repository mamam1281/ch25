1/2~1/6 배포 직후 홍보/유저 유치 운영 계획 (일자별 실행안)

작성일: 2026-01-02 적용 기간: 2026-01-02 ~ 2026-01-06 목표(북극성): CC 리텐션 유도(재방문/플레이 루프 유지 → Ticket Zero 전환 → 외부 CC 충전)

0) 전제/변경사항(이번 배포에서 무엇이 달라졌나)

0.1 핵심 UX 변경(유저가 체감하는 것)

  • 웰컴(NEW_USER) 모달이 전 유저에게 노출됨
  • 4개 미션 완료 전까지는 계속 뜸(영구 숨김/“다시 보지 않기” 제거)
  • /api/new-user/status는 대상자 필터(eligible gate)를 제거하여, 로그인 유저면 status/미션 정보를 제공

0.2 운영 리스크(홍보 문구에서 특히 주의)

  • 모달 상단 카피가 “10,000원 받기”로 고정 노출됨
    • 운영자는 NEW_USER 미션이 4개 존재하고, 각 미션 보상 합계가 카피와 충돌하지 않는지(합=10,000) 확인해야 함
  • 현금성 보상(포인트/코인/키 등)은 원칙적으로 수동 지급/검증이 필요한 구간이 존재함
    • 홍보에서 “즉시 지급/확정 지급”처럼 약속하면 CS 폭주/신뢰 붕괴 리스크

1) 1/2~1/6 운영 전략(요약)

1.1 목표별 운영 레버

  • Driver(재방문/플레이): 티켓/플레이 연료, 웰컴 미션(행동 유도)
  • Outcome(충전): Ticket Zero 시점의 CTA/링크/카피 점검, “충전하면 뭐가 달라지는지”를 과장 없이 명확히
  • Safety(신뢰/비용): 현금성 자동 약속 금지, 공지 1개로 통합(핀), 문의 대응 템플릿 고정

1.2 퍼널(유입→활동→전환) 한 줄 정의

  • 유입(텔레그램 공유 링크) → 첫 진입(랜딩) → 웰컴 모달(4 미션) → 게임 플레이 루프 → Ticket Zero → 외부 CC 충전

2) 매일 공통으로 하는 “최소 운영 루틴”(30~60분)

2.1 오전(또는 배포 직후) 10분: 상태 점검

  • (필수) NEW_USER 미션 4개가 활성화되어 있는지 확인(0개면 모달에 “미션 준비중”이 뜸)
  • (필수) 모달 카피(10,000원)와 실제 미션 보상 합(또는 지급 정책)이 충돌하지 않는지 확인
  • (필수) 텔레그램에서 진입했는데 /login 페이지가 보이는 재현이 있는지 1회 확인

2.2 낮 10~20분: 유입/공지 집행

  • 텔레그램 공지 1개만 유지(핀 고정)
  • 공유용 3줄 카피 + 링크 1개만 사용

2.3 저녁 10~20분: 회고(숫자 3개만)

  • Outcome: 전일 대비 충전건수/유니크(가능한 범위)
  • Safety: 수동지급 대기열/지급 지연 CS 키워드
  • Note: “오늘 바꾼 것” 1줄 기록

(참조) KPI 병렬 운영 SOP: 20260101_kpi_parallel_admin_excel_sop_ko.md

3) 일자별 상세 실행안

Day 1 — 1/2(금): “재오픈 공지 + 웰컴 미션 고정”

목표: 유저가 다시 들어오게 만들고(Driver), 웰컴 모달로 행동을 붙잡아 둔다.

  • 실행(오전/배포 직후)

    • 공지 1개 업로드 후 핀 고정(아래 카피 사용)
    • NEW_USER 미션 4개/보상 합/정책 확인
    • 웰컴 모달이 랜딩에서 뜨는지 1회 확인

  • 실행(오후)

    • 친구/그룹 공유: 3줄 카피 + 링크만 배포(과장 금지)
    • DM 문의는 템플릿으로만 응대(추가 약속 금지)

  • 실행(저녁)

    • CS 키워드 모니터링: “지급/먹튀/언제/10,000”
    • 필요 시 공지에 1줄 추가: “현금성 보상은 수동 확인 후 순차 처리”

1/2 공지 카피(핀용, 신뢰형)

  • 업그레이드 반영 완료했습니다. 지금부터 정상적으로 플레이 가능합니다.
  • 웰컴 미션(4개)을 완료하면 보상이 누적됩니다.(앱 내 안내 기준)
  • 현금성 보상(포인트/코인/키)은 자동이 아니며, 기록 확인 후 수동/순차 처리될 수 있습니다.
  • 문의는 본 공지 댓글로만 부탁드립니다(핀 고정).
  • 접속 링크: https://t.me/jm956_bot/ccjm

1/2 공유용 3줄(그룹/친구)

  • 업그레이드 끝났고 지금 안정적이에요.
  • 들어오면 웰컴 미션 4개로 바로 할 게 생겨요.
  • https://t.me/jm956_bot/ccjm

Day 2 — 1/3(토): “첫 주말 확대(공유/참여 유도)”

목표: 주말 유입을 최대화하되, 약속 리스크 없이 ‘가볍게 한 판’으로 진입 장벽을 낮춘다.

  • 실행(오전)

    • 공지 갱신(필요 시): “주말엔 플레이/티켓 중심”을 강조
    • 웰컴 미션 완료율(체감) 체크: 완료가 너무 어렵다면 미션 구성/난이도 조정은 ‘다음 배포’로 미루고, 카피로 기대치를 조절

  • 실행(오후)

    • 채널/그룹 1회 홍보(짧게): “한 판만 해도 감 잡힘” 프레임
    • 공유 링크는 1개만 유지(가능하면 startapp=ref_...로 추천 집계)

  • 실행(저녁)

    • DM 응대: “오늘 할 수 있는 것(플레이/미션)”만 안내

1/3 채널 홍보 카피(짧게)

  • 주말엔 한 판만 해도 감 잡혀요. 들어오면 웰컴 미션 4개가 바로 뜹니다.
  • 접속: https://t.me/jm956_bot/ccjm

Day 3 — 1/4(일): “완료 인증/사회적 증거(스크린샷 유도)”

목표: 신뢰를 만들기 위해 “완료했다/진행 중” 같은 사회적 증거를 만든다.

  • 실행(오전)

    • 공지 댓글에 “완료 스샷/닉네임” 남기면 확인해준다는 안내(단, 지급 약속 X)

  • 실행(오후)

    • 운영자(또는 고정 유저) 2~3명에게 부탁: 웰컴 모달/미션 진행 스샷 1장씩 공유
    • 공지에 ‘진행 예시’로 스샷 1~2장 업로드(가능한 채널이라면)

  • 실행(저녁)

    • Safety 체크: “10,000원” 표현으로 오해가 커지는지 확인
    • 오해가 크면 공지 문구를 “앱 내 보상 누적(수동 확인 가능)” 형태로 완화

1/4 공지 댓글 유도 문구(짧게)

  • 웰컴 미션 진행 중이면 스샷 1장만 댓글로 남겨주세요. 확인 도와드릴게요(지급 약속이 아니라 진행 확인 목적).

Day 4 — 1/5(월): “평일 루틴 고정(재방문 유도)”

목표: 주말에 들어온 유저가 월요일에 다시 오게 만든다.

  • 실행(오전)

    • 공지 갱신: “오늘도 한 판” 프레임(짧게)
    • 문의 폭주가 있으면 ‘큐 처리’ 원칙을 고지(실시간 대응 금지)

  • 실행(오후)

    • DM 템플릿으로만 응대: “오늘 할 것 1개”만 안내(미션/플레이)

  • 실행(저녁)

    • KPI 3개 기록 + 내일(1/6) 준비

1/5 공지 카피(초단문)

  • 월요일도 한 판만! 웰컴 미션 4개는 완료 전까지 계속 뜹니다.
  • 접속: https://t.me/jm956_bot/ccjm

Day 5 — 1/6(화): “공백/점검 모드 대비(신뢰 방어)”

목표: (문서 기준) 1/6~1/7 공백 가능성이 있어, 미리 기대치를 고정해 신뢰를 지킨다.

  • 실행(오전)

    • 공백이 실제로 발생하는지 내부 상태 확인
    • 공백 가능성이 있으면 공지에 “정산/점검 + 재개 시각 + 수동 지급 원칙” 3요소를 선반영

  • 실행(오후)

    • 공지 1개로만 유지(추가 공지 금지)
    • 문의는 댓글 큐로만 수집

  • 실행(저녁)

    • 복귀 체크리스트 초안 작성(1/7 또는 1/8 재개용)

1/6 공백 대비 공지(1줄 배너/상단용)

  • 정산/점검 진행 가능성: 현금성 보상은 수동 지급, 일정은 공지로 안내드립니다.

(참조) 공백 모드 런북: 20260101_cc_retention_gap_runbook_example_ko.md

4) 운영자가 매일 기록하는 1줄 로그(템플릿)

  • date: YYYY-MM-DD
  • what_changed: 공지/이벤트/설정 변경 1줄
  • outcome: 충전건수/유니크(가능 범위)
  • safety: 대기열/CS 키워드
  • note: 원인 추정 1줄

5) 금지사항(반드시 지키기)

  1. 확정할 수 없는데 “즉시 지급/오늘 지급” 약속하기
  2. 공지를 여러 개로 쪼개서 정보 분산시키기
  3. 현금성 보상을 자동 지급처럼 홍보하기
  4. 링크를 여러 개 뿌려서 문의/혼란 늘리기(링크는 1개)
  5. CS를 실시간으로 다 받으려다 운영 자체가 멈추기(댓글 큐로만)

6) 체크리스트(배포 당일 필수)

  • NEW_USER 미션 4개 활성화/보상 합 확인
  • 웰컴 모달이 랜딩에서 노출되는지 확인
  • Telegram initData 없는 접근에서 /login 노출이 있는지 확인(있으면 “텔레그램 접속” 안내 화면 노출이 맞는지 확인)
  • Ticket Zero CTA/링크 정상 동작 확인
