## 8. 지속 접속을 만드는 구조
### 8.1 Daily Quest (일일 접속 고리)
* **구조 목적:**
  * 매일 접속하지 않으면 손해라는 인식 형성
  * 소액 유저도 ‘할 일’을 만들어 이탈 방지
* **기본 퀘스트 설계:**
  * 오늘 룰렛 3판 → **즉시 보상 지급**
  * 오늘 팀 포인트 30점 달성 → **팀 기여도 + 개인 보상 트리거**
* **UI 노출 방식:**
  * 홈 화면 상단 고정 카드
  * 진행도 바로 채워지는 애니메이션 바 제공
* **행동 유도 카피 예시:**
  * “오늘 퀘스트 1개 남음”
  * “지금 1판만 더 하면 완료”
### 8.2 누적 미션 연계 (연속 접속 유도)
* Daily Quest 연속 달성 시:
  * 3일 연속 → 보너스 티켓 1장
  * 5일 연속 → 시즌패스 XP 부스터
* 핵심 설계 포인트:
  * 하루라도 놓치면 누적 초기화
  * **연속성 깨질 때 손실감(Loss Aversion) 유발**

## 9. 결론 (운영 철학)
이 프로젝트의 리텐션 핵심은
* **속도 (지금 당장 할 수 있어야 함)**
* **진행 (항상 다음 목표가 보여야 함)**
* **소속 (팀/랭킹에서 빠질 수 없게 만드는 구조)**

겉보기에는 전원 참여 이벤트지만,
실제 성과와 보상은 유효 이용자 중심으로 설계되어 있으며
무임승차는 시스템 내부에서 자연스럽게 제어된다.
