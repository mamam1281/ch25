# 프론트엔드 설계 검증 체크리스트

- 문서 타입: 프론트엔드 검증 체크리스트
- 버전: v1.1
- 작성일: 2025-12-10
- 작성자: 시니어 백엔드/FE 협업 가이드
- 대상 독자: 프론트엔드 개발자, QA, 테크 리드

## 1. 목적 (Purpose)
- 설계/구현 후 프론트 전반이 규칙에 맞게 준비되었는지 빠르게 점검하는 체크리스트를 제공한다.

## 2. 범위 (Scope)
- Scope: 라우트/스택/상태/UX/보안/빌드/테스트 및 UI 타겟 적합성 확인.
- Out of Scope: 백엔드 검증(별도 문서 참조).

## 3. 용어 정의 (Definitions)
- Checklist: 완료 여부를 확인하는 항목 목록.

## 4. 본문 (체크리스트)
- 라우트/네비게이션
  - [ ] `/` → today-feature → feature 페이지 리다이렉트가 구현되어 있다.
  - [ ] User/Admin 라우트 분리 및 보호 라우트가 적용되어 있다.
- 스택/상태
  - [ ] React 18/TS5/Vite6/Tailwind3/Query v5/Axios/RHF+Zod 버전이 적용되었다.
  - [ ] apiClient/adminApiClient가 분리되어 있다.
- UX/컴포넌트
  - [ ] 공통 버튼/모달/토스트 컴포넌트가 정의되어 있다.
  - [ ] 로딩/에러 패턴이 모든 주요 페이지에 적용되었다.
- 기능별 페이지
  - [ ] 룰렛/주사위/복권/랭킹/레벨 화면이 API 스펙과 일치한다.
  - [ ] SeasonPass 요약/결과 표시가 일관되게 배치되어 있다.
- Admin
  - [ ] 모든 설정 폼에 react-hook-form + Zod 검증이 적용되었다.
  - [ ] 룰렛 6칸, 복권 Σweight>0, ranking UNIQUE(date, rank) 검증을 UI에서 수행한다.
- 보안
  - [ ] 토큰 저장/401 처리/Protected Route 규칙이 반영되었다.
  - [ ] 민감정보 로그/URL 노출 금지 규칙을 준수한다.
- 빌드/배포
  - [ ] VITE_API_BASE_URL / VITE_ADMIN_API_BASE_URL 등 env가 환경별로 설정되었다.
  - [ ] dist 빌드 산출물이 배포 환경에서 정상 동작 확인되었다.
- 테스트/QC
  - [ ] 홈 리다이렉트, 각 게임 플레이, 레벨, Admin→User 반영이 테스트되었다.
  - [ ] 빈 상태/에러 상태/로딩 상태 QC 항목이 통과되었다.
- UI 타겟 적합성 (크리스마스 다크 카지노 톤, 20~60대 남성)
  - [ ] 색상/톤이 다크 + 크리스마스 악센트(그린/레드/골드) 조합을 유지한다.
  - [ ] 주요 CTA(스핀/주사위/복권/스탬프)가 1차 가시 영역에 위치하고 `text-lg` 이상 가독성을 확보한다.
  - [ ] 애니메이션/일러스트가 성인 카지노 톤을 유지하며 어린이풍 요소가 없다.
  - [ ] 결과 화면이 손실만 강조하지 않고 다음 행동(내일 재참여, 레벨 안내)을 제공한다.
  - [ ] `docs/07_review/frontend_christmas_theme_review_v1.0.md`의 검증 상태가 최신이며, 개선 액션이 필요 시 06/09번 문서에 반영되었다.
