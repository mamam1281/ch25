# 🧭 개발 로그: 보상함(보유함/상점) 탭 라우팅 + 하단 네비 추가

**작성일**: 2026-01-06  

---

## 1. 개요
모바일 하단 네비에 **보상함** 메뉴를 추가하고, 보상 UX를 **보유함(통합) ↔ 상점** 탭 라우팅으로 정리했습니다.

---

## 2. 변경 사항

### 2.1 하단 네비
- 순서: **홈 - 게임 - 금고 - 보상함 - 이벤트**
- 신규 메뉴: `보상함` → `/rewards`

### 2.2 라우팅
- 신규 라우트: `/rewards`
- 기존 `/inventory`는 `/rewards`로 리다이렉트
  - 금고/기타 진입 흐름은 유지하면서 보상함 통합 화면으로 합쳐짐

### 2.3 보유함 통합 화면
- 기존 인벤토리의 **보유 아이템 / 티켓 지갑**을 한 화면에 통합 표시
- 통합 화면에서는 기존 **다이아→상점 안내 카드**를 제거
- 대신 상단 `안내` 버튼으로 토스트 형태의 보조 설명 제공

### 2.4 상점 탭 헤더
- 상점 상단에 `보유함 / 상점` 라우팅 탭을 추가
  - `보유함` 클릭 시 `/rewards`로 이동

---

## 3. 검증 체크리스트
- `npm run build` 성공
- 하단 네비에서 `보상함` 진입 정상
- `/inventory` 진입 시 `/rewards`로 리다이렉트 정상
- `/rewards` ↔ `/shop` 탭 라우팅 정상

---

## 4. 리스크/메모
- 기존 `/inventory` URL을 외부에서 사용 중인 경우, 이제 `/rewards`로 이동합니다(동작은 동일 목적).
