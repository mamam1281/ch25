# Archive Policy

## 1) 보관 위치
- 폐기/구버전 문서는 `docs/archive/YYYYMM/`로 이동한다.
- YYYYMM은 아카이브 처리 월 기준으로 기록한다.

## 2) 아카이브 기준
- 상태 기준: 기능 종료, 정책 폐기, 운영 중단, Feature Sunset
- 버전 기준: 상위 버전 문서가 SoT로 확정된 경우
- 날짜 기준: 최신 업데이트가 오래되어 더 이상 운영/개발에 직접 사용하지 않는 경우

## 3) 처리 절차
1. 최신 SoT 문서에 대체 문서 링크와 전환 사유를 추가한다.
2. 아카이브 대상 문서 상단에 상태/전환일을 명시한다.
3. `docs/archive/YYYYMM/`로 이동하고, 기존 인덱스에서 제거한다.
4. 인덱스/변경로그에 아카이브 이동 사실을 기록한다.
