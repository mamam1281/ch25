# 12/25-01/01 연말 브릿지 시즌 상세 운영 기획안 (v1.0)

본 문서는 `2026` 프로젝트의 성공적인 연착륙을 위해 12/25부터 01/01까지의 운영 액션 플랜을 신규/기존 유저별로 상세히 기술합니다.

---

## 1. 운영 핵심 원칙
- **지급 차별화**: 내부 재화(티켓)는 **자동**, 외부 재화(포인트/코인) 및 주요 키는 **관리자 수동 지급**.
- **손실 회피(Loss Aversion)**: "안 하면 손해"라는 인식을 심어주는 카피와 보상 배치.
- **브릿지 전략**: 12월의 활동이 1월의 시드(Seed)와 3월의 대금고(Grand Vault)로 이어진다는 연속성 강조.

---

## 2. 유저 그룹별 상세 전략

### A. 신규 유저 (Target: 가입 7일 이내)
**"첫인상과 자산 형성"**
1.  **Welcome 적립**: 가입 즉시 시즌패스 레벨 1~3을 달성하게 하여 '룰렛/다이스 티켓' 획득 경험 제공.
2.  **금고 활성화**: 첫 게임 플레이 결과가 나오는 즉시 금고(Vault)에 최소 200원 이상 적립되는 모습을 UI로 노출.
3.  **성장 가이드**: "지금 10레벨을 찍으면 1월 시작 시 30,000원을 받고 시작한다"는 구체적인 정착 보상 제시.
4.  **트리거**: 티켓 소진(Ticket Zero) 시점에 "체험용 보너스" 대신 "금고에 쌓인 내 돈 확인" 모달을 띄워 입금 유도.

### B. 기존 유저 (Target: 기존 활동 유저 및 복귀 유저)
**"자산 보호와 락인(Lock-in)"**
1.  **자산 이월 알림**: "지난 시즌의 성과가 Gold/Diamond Vault에 예치되었습니다. 12/31까지 활동하지 않으면 소멸됩니다." 메시지 발송.
2.  **고액 보상 배치**: 레벨 9, 10에 파격적인 포인트(5만, 10만)를 배치하여 시즌 끝까지 완주 유도.
3.  **계급 의식**: 팀 배틀 승리 팀에게 1월 시작 시 보너스 토큰 지급을 예고하여 팀 기여도 유지.

## 4. 보상 지급 매뉴얼 (Operator Manual)

### 1) 자동 지급 대상 (시스템 처리)
- 룰렛 티켓, 다이스 티켓, 복권 티켓, 티켓 번들.
- 유저가 '보상 받기' 클릭 시 즉시 인벤토리에 반영됨.

### 2) 관리자 수동 지급 대상 (관리자 처리)
- **대상**: CC 포인트(1만~10만), Gold Key, Diamond Key.
- **절차**: 
    1. 어드민 '시즌패스 보상 로그' 확인.
    2. 유저 ID 및 보상 종류 확인.
    3. 외부 플랫폼(CC)에서 해당 유저에게 포인트 전송.
    4. 어드민에서 '지급 완료' 상태로 변경 (또는 메모 기록).

---

## 5. 긴급 대응 가이드 (Troubleshooting)

- **Mixed Content 에러 발생 시**: 브라우저 캐시 삭제 안내 또는 `src/admin/api/httpClient.ts`의 `origin` 설정 재확인.
- **시즌패스 미노출 시**: `season_pass_config` 테이블의 `is_active=1` 값과 날짜 범위(`start_date`, `end_date`) 재확인.
- **적립 누락 발생 시**: `VaultEarnEvent` 로그 테이블을 조회하여 멱등 키(ID) 충돌 여부 확인.

---

이 운영안은 유저의 **심리적 자산 가치**를 극대화하고, 운영자의 **수동 관리 리스크**를 최소화하도록 설계되었습니다.
